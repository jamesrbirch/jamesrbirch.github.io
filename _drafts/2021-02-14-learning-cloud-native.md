Learning Cloud Native

-   Setup a local CI/CD to test ideas without cloud hosting costs

The last point can be a contentious one, "The cloud is cheap!", "I host 8000 projects and it costs me 0.000001p per year".
Well yes and no, I have no problem paying for hosting for a project but the last few times I've made something it's been made in the "old ways" a big fat monolithic app with a SQL backend. I have a lot to learn and catch up on in cloud native land but I still have the need to make things and get them working. Something that is useful and works has value today whereas a perfectly architected and scalable web platform that take me a year to learn has no immediate benefit. To me this is the half way house and allows a more permanent exploratory path than just having docker on my dev machine.
